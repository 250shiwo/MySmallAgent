- **启动编排**：遵循 `Settings → LLMClient → MemoryManager → ToolRegistry → Agent → SessionManager → CLI` 的固定初始化顺序，确保各子模块在启动时获得所需依赖。
- **动态装配**：`create_default_registry` 作为工厂函数，根据可选的 `memory_manager` 和 `sessions_dir` 动态注册 `memory_save` 与 `session_search`，实现工具层的条件装配。
- **记忆注入**：`Agent` 在初始化时调用 `MemoryManager.load_memory_text()` 将长期记忆静态注入为第二条 system 消息，实现跨会话上下文感知。
- **渲染适配**：`CLI` 层根据 `Agent.streaming_enabled` 状态自动切换同步（`run_turn`）或异步生成器（`run_turn_stream`）模式，并统一调用 `SessionManager` 进行落盘。
- **存储契约**：所有子模块共享 `.genesis/` 目录下的文件存储约定（`memory.json` 与 `sessions/*.json`），由 `MemoryManager` 和 `SessionManager` 分别维护原子写与版本管理。
