- **启动编排**：`__main__.py` 作为唯一入口，按序初始化配置、LLM 客户端、记忆/会话管理器及技能注册表，并将依赖注入 Agent 与 CLI。
- **状态管理契约**：`SessionManager` 负责跨进程对话恢复，`MemoryManager` 负责跨会话偏好注入，两者均通过原子写策略保障数据一致性。
- **动态提示词组装**：`PromptManager` 在启动时拼接基础指令与自动发现的技能索引，保持 System Prompt 前缀稳定以优化缓存命中。
- **工具与技能扩展**：采用对称的 `ToolRegistry` 与 `SkillRegistry` 模式，支持通过 `dispatch` 机制实现组合工具（如 research_topic）及 LLM 自主激活的技能工作流。
