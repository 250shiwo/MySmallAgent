- 启动顺序固定为 Settings → LLMClient → MemoryManager → ToolRegistry → Agent → SessionManager → CLI，确保各子模块在初始化时获得所需依赖。
- `create_default_registry` 作为工厂函数，根据可选的 `memory_manager` 和 `sessions_dir` 动态注册 `memory_save` 与 `session_search`，实现工具层的条件装配。
- `Agent` 接收 `MemoryManager` 并在 `__init__` 中调用 `load_memory_text()` 注入第二条 system 消息，实现跨会话记忆的静态注入。
- `CLI` 层根据 `Agent` 的 `streaming_enabled` 状态选择同步或异步生成器模式（`run_turn` vs `run_turn_stream`）进行渲染，并统一调用 `SessionManager` 进行落盘。
- 所有子模块共享 `.genesis/` 目录下的文件存储约定（`memory.json` 与 `sessions/*.json`），由 `MemoryManager` 和 `SessionManager` 分别维护原子写与版本管理。
