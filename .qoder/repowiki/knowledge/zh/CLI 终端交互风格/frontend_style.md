该仓库是一个纯命令行（CLI）智能体应用，不包含任何 Web 前端代码（HTML/CSS/JS），因此不存在传统意义上的“前端样式”系统（如 CSS 方法论、组件库或设计令牌）。

其用户界面风格完全依赖于终端渲染，主要技术栈如下：

1. **核心库**：
   - `rich`：用于在终端中渲染富文本、Markdown、面板（Panel）和状态指示器。通过 `Console` 对象控制输出样式（如颜色、加粗、边框）。
   - `prompt-toolkit`：用于处理交互式输入提示（Prompt），提供自动补全和历史记录支持。

2. **视觉约定**：
   - **面板化输出**：使用 `rich.panel.Panel` 包裹欢迎信息、帮助信息和危险操作确认提示，通过 `border_style` 区分不同语境（如蓝色表示欢迎，黄色表示警告，绿色表示帮助）。
   - **状态反馈**：使用 `rich.status.Status` 显示“Thinking...”等异步加载状态。
   - **Markdown 渲染**：智能体的回复通过 `rich.markdown.Markdown` 直接渲染，保持 LLM 输出的原始格式。
   - **颜色语义**：
     - 蓝色 (`blue`)：常规信息/欢迎。
     - 黄色 (`yellow`)：警告/确认。
     - 绿色 (`green`)：成功/帮助。
     - 红色 (`red`)：错误/未知命令。
     - 青色 (`cyan`)：思考状态/命令高亮。

3. **开发约束**：
   - 所有 UI 逻辑集中在 `my_small_agent/cli.py`。
   - 不引入任何 Web 相关的样式文件。
   - 样式修改需通过 `rich` 提供的 API 进行，而非 CSS。
